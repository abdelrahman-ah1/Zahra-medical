PROJECT CODE EXPLANATION
========================

1. LOGIN PAGE (Login.html)
--------------------------
Frontend (Login.html, login.js):
- Structure: 
  - Contains two main forms: Sign In and Sign Up.
  - Includes a hidden "Change Password" form.
  - Uses a "split container" design where clicking "New here?" or "One of us?" buttons toggles a CSS class (sign-up-mode) to animate the switch between forms.
- Logic (login.js):
  - AJAX Submission: Submits forms to the backend (signin.php, signup.php) asynchronously without reloading the page.
  - Validation: Checks if passwords match during signup.
  - Security: Sets a local flag `localStorage.setItem('isLoggedIn', 'true')` upon successful login to track session state on the client side.
  - Change Password: Toggles the visibility of the change password form and sends a request to change_password.php.

Backend (signin.php, signup.php, change_password.php):
- Database connection via db_conn.php.
- Sign In: 
  - Verifies if the username exists in the `users` table.
  - Uses `password_verify()` to check the hashed password.
  - Starts a PHP session (`$_SESSION['user_id']`).
- Sign Up: 
  - Checks if the username or email already exists.
  - Hashes the password using `password_hash()` before inserting into the database.
- Change Password: 
  - Verifies the user's security question and answer before allowing a password reset.

2. DASHBOARD (dashboard.html)
-----------------------------
Frontend (dashboard.html, dashboard.js):
- UI: A sidebar navigation manages three views:
  1. Overview: Displays statistics cards.
  2. Reservations: A table of reservations.
  3. Feedbacks: A grid of customer reviews.
- Data Loading: 
  - `fetchReservations()` and `fetchFeedbacks()` are called on page load to retrieve data from the backend.
- Interactivity:
  - Status Update: Changing the dropdown in the reservations table sends a POST request to update the status (Pending/In Progress/Completed).
  - Delete: Buttons triggered to delete reservations or feedback entries.

Backend:
- reservation.php:
  - GET: Returns all reservations as JSON.
  - POST (JSON): Handles Admin actions (update_status, delete).
  - POST (Form Data): Handles public reservation form submissions (sanitizes inputs, inserts into DB).
- get_feedback.php: Returns all feedback rows as JSON.
- delete_feedback.php: Deletes a specific feedback entry by ID.

3. PROFILE (profile.html)
-------------------------
Frontend (profile.html, profile.js):
- Loading Data: 
  - On load, `profile.js` requests `get_profile.php`.
  - If the user is not logged in (session invalid), it redirects to Login.html.
- Display: 
  - Populates readonly input fields with user information (Name, Email, Phone, Security Question).
- Logout: 
  - Clears `localStorage` and redirects to `logout.php`.

Backend (get_profile.php):
- Checks if `$_SESSION['user_id']` is set.
- Fetches user details from the database using the session ID.
- Returns data as JSON.

4. AUTHENTICATION (auth.js)
---------------------------
Purpose: global script for session management on main pages (like index.html).
- UI Updates: 
  - Checks if the user is logged in via `localStorage` and `check_session.php`.
  - If logged in: Changes "Sign In" link to "My Profile".
  - If username is 'admin': Injects a "Dashboard" link into the navigation menu.
- Session Sync: 
  - Verifies session validity with the server.
  - If the PHP session is expired, it clears the local storage state to keep the UI in sync.
